# ゲーム内音声作成ツール 要件定義書 v0.3

## 1. 文書情報

- 文書名: ゲーム内音声作成ツール 要件定義書
- 版数: v0.3
- 作成目的: Unity上で動作するゲーム内音声作成ツールの要件を整理し、今後の仕様書・設計書作成の基礎資料とする
- 想定利用形態: UnityPackageとして配布し、GitHubにPublic公開する。配布用成果物はGitHub Releaseとして公開し、同一成果物をBOOTHでも出品する
- ライセンス: MIT

## 2. 背景

ゲーム開発では、UI操作音、アクション効果音、環境音、短いジングル、簡易BGMなど、多様な音声素材が必要となる。既存の外部ツールやDAWを利用する方法もあるが、ゲーム開発フローの中で軽量かつ迅速に音声を試作・編集・再生・出力できる環境をUnity内に持つことで、制作効率の向上が期待できる。

本ツールは、Unityエディタ上で完結してゲーム内音声を作成し、再編集可能な編集データと音声ファイルを出力できることを目的とする。

## 3. 目的

本ツールの目的は以下の通りとする。

- Unity上でゲーム内音声を作成できること
- GUI操作により音を構成し、試聴・編集・保存できること
- 効果音だけでなく、複数音を並べた短い音楽的アウトプットにも対応すること
- 編集用データを保存し、後から再編集可能であること
- Unityプロジェクト内または任意フォルダへ音声ファイルを出力できること
- 将来的な機能拡張に耐えられる構造を持つこと
- UnityPackageとして第三者へ配布可能な品質・構成を持つこと
- GitHub ReleaseおよびBOOTH出品を前提とした配布物構成を持つこと

## 4. 対象範囲

### 4.1 対象とする音

本ツールが対象とする音は、**ボイスを除くゲーム内音声全般**とする。

対象例:

- UI効果音
- アクション効果音
- システム音
- 環境音風の短音素材
- ジングル
- 短いループBGM
- 簡易な楽器音
- 和音を含む音素材

### 4.2 対象外

初期リリースでは以下は対象外とする。

- 人の声の合成
- 歌声合成
- ボイス編集
- 立体音響の本格対応
- 範囲再生
- 外部DAWレベルの高度なミキシング
- 超写実的な生楽器音源の再現
- MP3等の追加音声形式出力

## 5. 想定ユーザ

本ツールの想定ユーザは以下とする。

- Unityを利用するゲーム開発者
- 効果音や簡易音楽をUnity上で作成したい開発者
- UnityPackageとして導入し、プロジェクト内で手軽に音を制作したい利用者
- GitHub公開物や配布物として導入する外部利用者
- BOOTH等の配布プラットフォーム経由で導入する利用者

そのため、本ツールは**自分専用ツールではなく、他の開発者にも利用されることを前提とした品質・操作性・配布物整備**を考慮する。

## 6. 動作環境

- 実行環境: Unity Editor
- 対応Unityバージョン: Unity 6000以降
- 対応OS: Windows 11
- ネットワーク要件: オフライン完結必須

補足:

- 将来的に他OS対応の可能性はあるが、初期要件としてはWindows 11を対象とする
- オフライン環境でも音声作成・再生・保存・出力が可能であること

## 7. システム概要

本ツールは、Unity Editor上で動作する音声作成・編集ツールであり、以下の機能群から構成される。

- 音の基本要素をGUIで設定する編集機能
- 複数トラック上にノートブロックを配置する編集機能
- 単発再生・ループ再生を行う試聴機能
- 編集データを保存・読み込みする機能
- 音声ファイルを出力する機能
- 設定値を保持するコンフィグ機能
- Undo / Redo機能
- 配布用サンプル・マニュアル・利用規約を含むリリース構成

## 8. 機能要件

### 8.1 音源生成機能

#### 8.1.1 MVPで対応する音源方式

初期リリースでは以下を対応対象とする。

- 波形合成
- ノイズ生成
- ADSRによる音量エンベロープ制御
- 簡易エフェクト

#### 8.1.2 ADSRの定義

本ツールにおけるADSRは以下の4要素を持つ音量エンベロープ制御とする。

- Attack: 音の立ち上がり時間
- Decay: 最大音量から持続音量へ移る時間
- Sustain: 音を維持する際の音量レベル
- Release: 音を止めた後に減衰して消える時間

#### 8.1.3 波形関連

初期リリースでは、波形合成によりゲーム用途の基本音を作成可能とする。具体的な波形の種類やパラメータ詳細は仕様書で定義する。

#### 8.1.4 ノイズ生成

初期リリースでは、打撃音・衝突音・短いSE等に利用可能なノイズ生成機能を持つこと。

### 8.2 簡易エフェクト機能

初期リリースで対応する簡易エフェクトは以下とする。

- ボリューム
- パン
- ピッチ
- フェードイン
- フェードアウト
- 簡易ディレイ

各エフェクトの詳細パラメータ、適用順序、制約は仕様書で定義する。

### 8.3 編集機能

#### 8.3.1 編集単位

本ツールは以下の編集単位をサポートすること。

- 1音単位での編集
- 複数音の配置
- 複数トラックでの編集
- 同時発音
- 和音

#### 8.3.2 配置方式

編集UIでは、**ノートブロックを時間軸上に配置する方式**を採用する。

#### 8.3.3 入力補助

GUI操作に加えて、**数値入力による補助入力**を可能とする。

対象例:

- 音程
- 長さ
- 開始位置
- 音量
- ADSR値
- エフェクト値

#### 8.3.4 トラック編集

複数トラックを扱えること。トラックごとの詳細制御項目は仕様書で定義する。

#### 8.3.5 同時発音・和音

同一タイミングに複数ノートを配置可能であり、和音表現に対応すること。

### 8.4 再生機能

本ツールは、編集した内容をUnity Editor上で試聴可能とする。

#### 8.4.1 対応する再生方式

- 単発再生
- ループ再生

#### 8.4.2 再生カーソル

タイムライン上に**再生カーソルを表示**し、現在再生位置を確認可能とする。

#### 8.4.3 将来対応

以下は初期リリース対象外とし、将来拡張とする。

- 範囲再生
- 高度な再生制御

### 8.5 保存・読込機能

#### 8.5.1 編集データ保存

再編集可能な編集用データを保存できること。

#### 8.5.2 編集データ読込

保存済みの編集用データを読み込み、編集・再生・出力できること。

#### 8.5.3 データ形式

編集用データは、**拡張性の高い構造化データ形式**とする。初期リリースでは **JSONを正式対応形式** とする。

#### 8.5.4 将来拡張

以下は将来拡張候補とする。

- XMLのインポート / エクスポート
- その他形式への対応
- 外部エディタで編集した構造化データの取り込み拡張

### 8.6 音声出力機能

#### 8.6.1 出力形式

初期リリースでは **WAV出力必須** とする。

#### 8.6.2 チャンネル構成

以下を選択可能とする。

- モノラル
- ステレオ

#### 8.6.3 出力先

音声ファイルは任意フォルダへ出力可能とする。

#### 8.6.4 出力先設定

出力先フォルダは毎回都度選択ではなく、**コンフィグで設定可能**とする。

#### 8.6.5 Unityアセット反映

Unityプロジェクト内への出力時、自動リフレッシュの有無をコンフィグで設定可能とする。

#### 8.6.6 初期対象外

- MP3出力
- 立体音響対応

### 8.7 コンフィグ機能

#### 8.7.1 管理単位

コンフィグは以下の二層構成とする。

- ツール共通設定
- プロジェクト単位設定

#### 8.7.2 管理対象例

対象詳細は仕様書で定義するが、少なくとも以下を含める。

- 出力先フォルダ
- 自動リフレッシュ設定
- 初期出力設定
- 表示関連設定
- 再生関連設定の一部

### 8.8 Undo / Redo機能

本ツールはUndo / Redoに対応すること。対象操作の範囲や履歴保持数などの詳細は仕様書で定義する。

### 8.9 利用者向け補助機能

UnityPackageとして他の開発者が扱うことを前提に、以下を考慮する。

- GUIベースで直感的に操作できること
- 読み込み失敗時に適切な通知を行うこと
- 保存失敗時に適切な通知を行うこと
- サンプルファイルを同梱すること
- マニュアルを同梱すること
- 利用規約を同梱すること

#### 8.9.1 サンプルファイル要件

- 基本操作の確認に利用できるサンプルを含むこと
- 読込・編集・再生・書き出しの確認に利用できること
- 少なくとも効果音系サンプルを含むこと
- 必要に応じて簡易音楽系サンプルも含められる構成であること

#### 8.9.2 マニュアル要件

マニュアルには少なくとも以下を含めること。

- 導入方法
- 基本操作
- 保存方法
- 読み込み方法
- 書き出し方法
- コンフィグ設定方法
- 制約事項
- 初歩的なFAQまたはトラブルシュート

#### 8.9.3 利用規約要件

利用規約には少なくとも以下を含めること。

- 利用条件
- 禁止事項
- 免責事項
- 著作権・再配布条件
- サポート範囲
- 規約改定に関する扱い

補足:

- 利用規約の内容は、本ツールの採用ライセンスと矛盾しないこと
- BOOTH出品時に、購入前確認可能な形で提示できる構成であること

## 9. 非機能要件

### 9.1 オフライン完結

- ネットワーク接続なしで利用可能であること
- 音声生成、編集、再生、保存、出力の一連の処理がローカルで完結すること

### 9.2 拡張性

- 将来的な機能追加に耐えられる構造を持つこと
- JSONを基盤に、将来XMLなどへの拡張が可能であること
- 外部エディタで編集したデータの取り込み拡張に配慮した構造とすること

### 9.3 配布性

- UnityPackageとして配布可能な構成であること
- GitHub Public公開を前提に整理された構成を持つこと
- 配布用成果物はGitHub Releaseとして公開可能であること
- BOOTHでは、GitHub Releaseとして公開する成果物と同一内容を出品できる構成であること
- 配布用リリースには、ツール本体に加えてサンプルファイル、マニュアル、利用規約を含めること
- BOOTH出品時に、利用規約を**購入前に確認できる状態**にすること

### 9.4 保守性

- 機能追加や仕様変更を考慮した構造を持つこと
- データ形式やコンフィグ形式が可読性・管理性を持つこと

### 9.5 ライセンス

- 本ツール本体はMITライセンスで公開可能であること
- 外部ライブラリ導入時はライセンス整合性を確認すること
- 配布時の利用規約や注意事項は、採用ライセンスと整合する内容であること

### 9.6 性能

- 通常的なゲーム開発用途において、編集・再生・保存・出力が実用上支障ない速度で行えること
- 詳細な性能基準は仕様・設計段階で定義する

### 9.7 再現性

- 同一編集データから同一条件で再出力した場合、再現可能な結果を得られること
- 小数演算や内部処理誤差の扱いは仕様で定義する

## 10. 将来拡張要件

- XML正式対応
- 外部エディタ連携強化
- MP3等の追加出力形式
- 範囲再生
- 立体音響対応
- より高度なエフェクト
- 高度なトラック編集機能
- より豊富な音源方式
- 高度な可視化UI
- サンプル音源ベースの楽器対応
- 他OS対応

## 11. 制約事項

- 初期リリースはUnity Editor上で動作すること
- 対応OSはWindows 11とする
- オフライン完結を満たすこと
- 初期出力形式はWAVのみとする
- ボイス機能は対象外とする
- 立体音響は初期対象外とする

- Unity上でツールを起動できる

- 音程、長さ、音量、ADSR等を編集できる
- 複数トラックに配置できる
- 同時発音と和音を扱える
- 単発再生・ループ再生できる
- 再生カーソルが表示される
- JSONで保存・読込できる
- WAVで書き出せる
- モノラル / ステレオを選択できる
- 出力先をコンフィグで管理できる
- Undo / Redoが利用できる

- 配布物をGitHub Releaseとして公開できる
